# 本次目标

- 将请求历史列表的响应摘要提取下沉到 SQLite 查询阶段，减少首次进入历史页时在应用层解析整段响应 JSON 的开销。

# 核心改动

- 为 `RequestHistoryEntity` 增加列表摘要字段，承接仓储层查询投影结果。
- 调整 `RequestHistoryRepository.GetHistoryAsync`，直接通过 SQLite `json_extract` 读取状态码、耗时、响应大小，并在查询阶段计算 `HasResponse`。
- 简化 `RequestHistoryService.GetHistoryAsync`，列表摘要不再解析 `response_snapshot_json`，仅详情加载时反序列化完整响应体。
- 新增服务单测与 SQLite 仓储集成测试，覆盖“服务使用投影字段”和“仓储正确提取摘要”两条链路。

- 当前摘要投影兼容 `camelCase` / `PascalCase` 两种 JSON 属性名；若后续序列化字段再次调整，需要同步更新 SQLite JSON 路径。
