# 本次目标
- 将 Swagger 导入接口从临时展示改为真实的本地 HTTP 接口。
- 让项目重新打开后，导入接口仍能在左侧接口树和右侧编辑区正常使用。

# 核心改动
- 为 `IRequestCaseService` 增加 `SyncImportedHttpInterfacesAsync`，按 `method + path` 识别导入接口并同步到 `request_cases`。
- 在 `RequestCaseService` 中实现导入接口同步：保留仍存在的接口 ID，更新名称/分组/描述，移除已不存在的导入接口。
- 在 `ProjectTabViewModel` 中移除导入接口临时树节点逻辑，改为在项目加载和导入成功后自动同步本地接口，再复用现有接口树与编辑区加载链路。
- 补充 ViewModel 和仓储测试，覆盖“初始化自动同步导入接口”和“按 method + path 覆盖更新导入接口”。

# 风险或遗留项
- 当前导入接口同步为“受 Swagger 导入管理”的本地 HTTP 接口，再次导入同一路径接口时会用最新 Swagger 内容覆盖接口定义。
- 已保存在这些接口下的请求用例会继续保留；仅当导入接口本身被移除时，关联用例才会被一并清理。
