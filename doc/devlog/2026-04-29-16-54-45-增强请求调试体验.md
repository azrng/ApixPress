# 增强请求调试体验

## 本次目标
- 增加请求手动取消能力。
- 优化请求失败、超时、取消以及 HTTP 错误响应的展示。
- 增加变量替换后的发送预览，降低环境变量和 BaseUrl 排查成本。
- 补充导入忙碌取消、未保存标签关闭保护和响应正文搜索/格式化。

## 核心改动
- 请求 loading 覆盖层增加“取消请求”按钮，并接入当前请求的取消令牌。
- `ResponseSectionViewModel` 增加响应通知、Body 搜索统计，并补充 XML 格式化。
- 请求编辑区展示最终发送预览，包含变量替换、BaseUrl 拼接、路径参数和查询参数后的 URL。
- Swagger/项目数据包导入忙碌覆盖层增加“取消导入”按钮。
- 工作标签增加未保存标记，关闭未保存标签时第一次拦截提示，第二次才放弃修改。
- 补充请求取消、响应错误提示、正文搜索、XML 格式化和未保存标签关闭保护测试。
