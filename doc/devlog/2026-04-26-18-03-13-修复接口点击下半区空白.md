# 修复接口点击下半区空白

## 背景
- 点击左侧接口后，顶部工作标签和请求行已经显示，但下方 Params、Body、Headers 与响应区域未渲染。

## 变更
- 先尝试为 `RequestEditorWorkspaceView` 内部的 `RequestEditorContentHostView` 显式传递当前 `ProjectTabViewModel` 数据上下文，但运行反馈仍未恢复。
- 进一步将请求编辑器下半区改为直接挂载 `QuickRequestWorkbenchView`、`HttpInterfaceWorkbenchView` 与 `HttpDocumentWorkspaceView`，复用各视图原有的可见性绑定切换内容。
- 绕开动态 `ContentControl` 挂载层，避免请求行已显示但下半区内容宿主未挂载导致空白。

## 验证
- `dotnet build .\src\ApixPress.App\Apifox.App.csproj -p:OutputPath=.artifacts\verify-app\`
- `dotnet test .\tests\ApixPress.App.Tests\ApixPress.App.Tests.csproj -p:OutDir="$PWD\.artifacts\verify-tests\"`

## 备注
- 本次为单点 XAML 绑定修复；建议在运行中的界面再次点击左侧接口，确认下方参数与响应区恢复显示。
