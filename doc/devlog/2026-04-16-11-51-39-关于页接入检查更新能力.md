- 在设置中心“关于”页接入真实检查更新能力，使用 GeneralUpdate 的 OSS 模式完成客户端升级。

# 核心改动

- 新增 `IApplicationUpdateService` / `ApplicationUpdateService`，负责拉取更新清单、比较版本并启动 `GeneralClientOSS`。
- 新增 `ApixPress.Updater` 项目，作为独立升级器调用 `GeneralUpdateOSS.Start()`。
- 关于页移除 mock 更新提示，改为展示真实最新版本、更新通道和检查状态。
- 新增 `VERSION` 与 `.github/workflows/release.yml`，发布时生成 zip 包和 `versions.json`。

# 风险或遗留项

- 当前仅补齐了 Windows `win-x64` 发布与更新链路，其他平台暂未接入。
- `GeneralUpdate` OSS 模式会在应用目录下载并解压更新包，若用户将程序放在无写权限目录，更新可能失败。
- GitHub Actions 需仓库实际存在 `azrng/ApixPress` 公开 release 权限与默认 `main` 分支，才能让默认清单地址直接生效。
