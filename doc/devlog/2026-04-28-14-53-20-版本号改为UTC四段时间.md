# 版本号改为 UTC 四段时间

## 本次目标
- 将自动生成版本号从本地日期改为 UTC 年月日小时四段格式，避免美国 CI 与本地时区差异导致版本不一致。

## 核心改动
- `Directory.Build.props` 默认版本改为 `DateTime.UtcNow` 生成 `yyyy.M.d.HH`。
- GitHub Actions 发布元数据改为用 UTC 时间生成同样的四段版本号。
- 更新设计文档中的版本规则说明。

## 修改文件
- `Directory.Build.props`
- `.github/workflows/release.yml`
- `doc/design/设计文档.md`
- `TASK.md`

## 校验情况
- 已执行：`dotnet msbuild src/ApixPress.App/Apifox.App.csproj -getProperty:Version -getProperty:InformationalVersion -getProperty:AssemblyVersion -getProperty:FileVersion`
- 结果：版本属性均为四段 UTC 格式，例如 `2026.4.28.06`。
- 已执行：`dotnet build Apifox.slnx`
- 结果：构建通过，0 警告，0 错误。

## 风险或遗留项
- 版本号小时展示为 UTC 小时，不再等同北京时间或 CI 所在地时间；这是本次为保证跨环境一致性做出的约定。
