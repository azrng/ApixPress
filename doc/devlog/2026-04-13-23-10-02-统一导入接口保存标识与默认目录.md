# 本次目标
- 修复导入接口在右侧编辑后再次保存会丢失导入标识的问题。
- 统一导入接口与手工新建 HTTP 接口的默认目录行为。

# 核心改动
- 在 `RequestWorkspaceTabViewModel` 中增加 `SourceEndpointId`，让接口标签在编辑和保存过程中持续保留原始 `EndpointId`。
- 调整 `BuildSnapshot`，保存接口或用例时不再把 `EndpointId` 清空，避免导入接口再次导入后生成重复本地接口。
- 将 HTTP 接口默认目录统一为 `默认模块`，并同步更新项目页当前目录回显。
- 补充 ViewModel 测试，覆盖“导入接口保存后仍保留 Swagger 标识”和“新建 HTTP 接口默认目录”。
