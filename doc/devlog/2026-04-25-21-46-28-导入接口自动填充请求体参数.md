# 导入接口自动填充请求体参数

## 本次目标
- 导入接口存在 JSON requestBody 时，打开详情自动切到 JSON 并填充示例参数。
- 导入接口存在 form-data 或 x-www-form-urlencoded requestBody 时，打开详情自动切到对应表单模式并带出字段名。

## 核心改动
- OpenAPI 解析增加 requestBody 模式识别，支持 `application/json`、`multipart/form-data`、`application/x-www-form-urlencoded`。
- JSON requestBody 在缺少 example/examples 时，可根据 schema properties 生成默认示例 JSON。
- 表单 requestBody 根据 schema properties 生成表单字段模板。
- `api_endpoints` 增加 `request_body_mode` 字段，并在数据库初始化时为已有库补列。
- 接口详情和导入保存链路按 `RequestBodyMode` 恢复 Body 选项与表单字段。

## 风险或遗留项
- JSON schema 示例生成目前覆盖常见 object/array/string/number/boolean/default/example/enum 和本地 `$ref`，复杂组合 schema（如 oneOf/allOf）暂未展开。
- 本地运行中的 `ApixPress.exe` 会锁定默认输出目录，因此验证命令使用隔离 artifacts 输出目录。
