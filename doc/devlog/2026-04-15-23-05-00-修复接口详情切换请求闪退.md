# 修复接口详情切换请求闪退

## 本次目标

- 修复进入接口详情后切换其他 HTTP 请求时应用闪退的问题。

## 核心改动

- 将 `ProjectTabViewModel.VisibleWorkspaceTabs` 从按次动态 `ToList()` 改为稳定的只读可观察集合，避免顶部 `TabStrip` 在切换请求时反复更换 `ItemsSource`。
- 在工作区标签集合和标签可见性变化时统一同步可见标签集合，保持请求切换过程中的选中项与标签容器状态稳定。
- 补充“连续加载两个 HTTP 接口并切换”回归测试，覆盖本次闪退场景。
