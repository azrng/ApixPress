# 响应提示与返回响应同一行展示

## 本次目标

- 将请求响应区中的提示信息收进“返回响应”标题同一行展示，并同步 HTTP 接口与快捷请求两处工作台布局。

## 核心改动

- 将 HTTP 接口工作台响应区从“标题 + 独立提示卡 + 内容区”的三行结构调整为“标题/提示/摘要 + 内容区”的两行结构。
- 为快捷请求工作台同步相同的响应头部布局，避免两套工作台的响应区行为再次分叉。
- 新增 `HttpResponseNoticeInlineCard` 样式，复用现有提示卡底色和边框，仅收紧内边距与垂直对齐，适配同一行展示。
