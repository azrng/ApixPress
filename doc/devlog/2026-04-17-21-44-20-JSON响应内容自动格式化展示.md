# 本次目标

- 当返回响应的 `Content-Type` 为 JSON 时，自动将响应体格式化后展示，避免整行压缩导致阅读困难。

# 核心改动

- 在 `ResponseSectionViewModel` 中增加响应体格式化逻辑。
- 当响应头为 `application/json`、`text/json` 或 `+json` 媒体类型时，尝试将响应体按缩进 JSON 重新序列化后展示。
- 若响应体不是合法 JSON，则保留原始内容，避免误伤异常返回。
- 补充 ViewModel 单测，覆盖标准 JSON、`problem+json` 和非法 JSON 回退场景。

# 风险或遗留项

- 当前仅按响应头自动识别 JSON；若后续需要支持无 `Content-Type` 但正文明显为 JSON 的场景，可再补充更宽松的兜底识别策略。
