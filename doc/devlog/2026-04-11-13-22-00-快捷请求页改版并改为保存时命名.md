# 快捷请求页改版并改为保存时命名

## 本次变更
- 将快捷请求页面单独调整为更接近参考图的布局：上方 URL 行，中间横向配置标签，下方返回响应区。
- 快捷请求不再常驻显示“请求名称”输入框，改为点击保存时弹窗输入名称和备注。
- 保留 HTTP 接口页现有独立布局，不再和快捷请求共用同一套展示结构。

## 关键实现
- 快捷请求编辑区支持 `Params`、`Body`、`Headers` 三个真实配置标签，并补充 `Cookies`、`Auth`、`前置操作`、`后置操作`、`设置` 占位入口。
- 保存快捷请求时，会弹出命名对话框，确认后才真正写入左侧快捷请求目录。
- 已保存的快捷请求再次保存时，会自动回填当前名称和备注，便于覆盖更新。
