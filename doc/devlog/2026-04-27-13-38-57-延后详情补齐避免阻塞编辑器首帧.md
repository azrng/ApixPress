# 延后详情补齐避免阻塞编辑器首帧

## 本次目标
- 继续压缩点击接口后的首屏等待感，避免详情反序列化和详情回填挤占编辑器首帧显示。

## 核心改动
- `RequestCaseService.GetDetailAsync` 将详情 DTO 反序列化放到后台线程执行，减少 UI 线程直接参与 JSON 解析。
- `ProjectWorkspaceCatalogViewModel.LoadWorkspaceItemDetailAsync` 在详情补齐前先 `Task.Yield()`，至少让编辑器首帧先完成一次调度再开始详情回填。
- 同步更新相关测试断言，适配“详情加载异步后移一拍”的新行为。
