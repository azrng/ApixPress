# 本次目标

- 压缩 HTTP 接口调试页的头部与参数区域，放大返回响应区域。

# 核心改动

- 将 HTTP 调试工作区上下主区域比例从 `3:2` 调整为 `2:3`，让响应区高于参数区。
- 为参数配置卡片增加独立高度上限，避免参数区域继续把响应区挤小。
- 收紧顶部模式栏、接口名输入、方法选择、BaseUrl、路径输入和发送/保存按钮的内边距与最小高度，减少头部占用。
- 提高 HTTP 响应卡片的最小高度，改善返回结果阅读空间。
