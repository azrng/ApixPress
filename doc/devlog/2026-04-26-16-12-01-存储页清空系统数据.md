# 存储页清空系统数据

## 背景
- 设置中心存储页的危险操作需要从“清空历史数据”调整为“一键清空系统数据”，二次确认后清空所有项目配置。

## 变更
- 新增 `ISystemDataService` / `ISystemDataRepository`，通过事务按依赖顺序清空项目、接口、请求参数、用例、环境变量、环境和请求历史表。
- 设置中心存储页危险区域改为“一键清空系统数据”，确认弹窗明确提示会删除项目、接口、用例、环境配置和请求历史记录。
- 清空成功后同步关闭已打开项目标签，清空首页项目列表与历史面板，让界面回到空项目状态。
- 更新 README 中存储页危险操作说明。

## 验证
- `dotnet build .\src\ApixPress.App\Apifox.App.csproj -p:OutputPath=.artifacts\verify-app\`
- `dotnet test .\tests\ApixPress.App.Tests\ApixPress.App.Tests.csproj -p:OutDir="$PWD\.artifacts\verify-tests\"`

## 备注
- 直接运行默认 `dotnet test` 时，当前正在运行的 `ApixPress.exe` 锁定默认 Debug 输出目录，因此改用独立输出目录完成测试验证。
