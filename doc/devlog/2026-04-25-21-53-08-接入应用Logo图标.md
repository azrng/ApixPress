# 接入应用 Logo 图标

## 本次目标
- 将生成的项目 Logo 接入主项目。
- 使用 `.ico` 作为 Windows 程序图标，同时保留 `.png` 作为应用内展示资源。

## 核心改动
- 从生成图片复制 `AppLogo.png` 到应用资源目录。
- 基于同一图片生成包含 16、24、32、48、64、128、256 尺寸的 `AppIcon.ico`。
- 在项目文件中配置 `ApplicationIcon`，用于 exe、任务栏和系统图标。
- 在主窗口配置 `Icon`，用于 Avalonia 窗口图标。

## 修改文件
- `src/ApixPress.App/Assets/AppLogo.png`
- `src/ApixPress.App/Assets/AppIcon.ico`
- `src/ApixPress.App/Apifox.App.csproj`
- `src/ApixPress.App/MainWindow.axaml`

## 校验情况
- `dotnet build src\ApixPress.App\Apifox.App.csproj --artifacts-path D:\GitHub\ApixPress\.test-output\icon-build "-p:DefaultItemExcludesInProjectFolder=$(DefaultItemExcludesInProjectFolder)%3B**\.artifacts\**"`：通过。

## 风险或遗留项
- 本轮未启动应用做真实窗口图标目视检查；构建已验证项目图标与 XAML 引用可编译。
