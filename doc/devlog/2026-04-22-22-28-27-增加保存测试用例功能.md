# 增加保存测试用例功能

## 本次目标
- 在 HTTP 接口工作区增加“保存测试用例”入口，并确保保存后能立即出现在左侧对应接口节点下。

## 核心改动
- 为 `MainWindowViewModel` 新增 `SaveHttpCaseCommand`，把界面按钮直接接到现有 `Workflow.SaveHttpCaseAsync()`。
- 在 HTTP 接口编辑区增加“用例名称”输入框和“保存测试用例”按钮。
- 在 HTTP 文档预览页右上角增加测试用例操作卡片，补齐用户当前视图下的直接入口。
- 修正 `SaveHttpCaseAsync()` 流程：保存用例前先把父接口同步插入目录，避免新建接口时只保存到库里但左侧树不显示。
- 补充 `ProjectTabViewModelTests.SaveHttpCaseAsync_ShouldCreateCaseUnderInterfaceNode`，验证用例会挂到对应接口节点下。
