# 需求文档

> 本文件用于承载项目的业务需求事实，不记录设计推演、实现过程或开发日志。

---

## 1. 项目背景

- 项目名称：API 协作平台
- 业务背景：需要一个本地可运行的 API 调试和管理工具，风格参考 ApixPress，降低接口导入、调试和用例复用的成本。
- 目标用户：API 开发人员、测试人员、接口联调人员。
- 立项原因：现有需求聚焦于 Swagger/OpenAPI 文档导入、HTTP 请求调试、调试用例保存与复用，适合作为桌面端高频工具落地。

## 2. 目标与范围

### 2.1 本期目标

- 支持导入 Swagger / OpenAPI 文档，并以树形结构管理接口。
- 支持配置并发送 HTTP 请求，查看响应结果与网络信息。
- 支持将请求保存为本地用例，并进行快速复用。
- 支持多项目管理，并在不同项目之间隔离接口文档、环境配置与调试资产。
- 支持每个项目维护多个环境，为环境配置独立 `BaseUrl` 与自定义变量参数。

### 2.2 本期范围

- Swagger / OpenAPI JSON 文档导入。
- URL 导入、本地上传、拖拽上传。
- 多项目创建、切换、重命名、删除与默认项目管理。
- 接口树、分组展示、搜索筛选。
- HTTP 请求参数编辑、请求发送、响应查看。
- 项目级多环境配置与变量管理。
- 本地用例保存、分组管理、快速加载与复制。
- 本地 SQLite 存储与基础备份方案。

### 2.3 非本期范围

- 团队在线协作与共享工作区。
- 复杂权限体系与组织管理。
- 云端同步、账号系统、远程备份。
- 高级自动化测试编排与性能测试。

## 5. 核心业务流程

### 5.1 主流程

1. 用户创建项目或进入已有项目。
2. 用户在当前项目下选择目标环境，系统加载该项目的 `BaseUrl` 与变量参数。
3. 用户导入 Swagger / OpenAPI 文档。
4. 系统解析接口分组、路径、参数、响应和数据模型。
5. 用户从接口树中选择目标接口，并在调试面板中补全参数。
6. 用户发送请求并查看响应状态、耗时、大小和响应内容。
7. 用户将当前请求保存为本地用例，用于后续快速加载和复制。

### 5.2 关键状态流转

- 初始状态：未创建项目时提示创建项目；项目存在但未导入文档时展示导入入口。
- 处理中：导入解析中、请求发送中、用例保存中。
- 成功态：项目保存成功、环境保存成功、文档解析成功、请求返回成功、用例保存成功。
- 失败态：项目名冲突、环境配置不完整、文档格式错误、请求异常、存储失败、解析失败。

## 8. 边界场景与异常

- 空数据时：未导入任何文档时，展示引导页和导入入口；无用例时展示空状态。
- 首次使用时：未创建任何项目时，展示“创建项目”引导，并禁用发送请求等依赖项目上下文的操作。
- 数据冲突时：保存同名用例时需提示覆盖、另存为或取消。
- 项目/环境冲突时：同一层级下项目名、环境名重复时需明确提示，避免隐式覆盖。
- 权限不足时：若后续接入系统权限或文件访问受限，应提供明确错误提示。
- 网络/服务异常时：请求失败需展示错误信息、请求详情和可重试入口。
- 用户误操作时：删除或覆盖用例前需要二次确认，并允许取消。
- 删除项目时：需提示会影响该项目下的接口文档、环境、用例和历史记录，并支持取消。
- 导入异常时：文档格式不合法、文件过大、URL 不可访问时，需给出明确失败原因。

## 9. 非功能要求

- 性能要求：Swagger 文档导入需支持 20MB 以内 JSON 文件；常规请求调试反馈需清晰可感知。
- 安全要求：本地请求调试需避免明文泄露敏感配置；SSL 忽略仅限开发环境使用。
- 审计要求：本期可不做完整审计，但至少保留基础请求日志和最近使用记录。
- 兼容性要求：桌面端需适配主流桌面环境，并保证常见窗口尺寸下可用。
- 部署/环境要求：支持桌面端单文件发布，本地数据优先落到 SQLite，并支持基础备份。
- 数据隔离要求：不同项目的环境配置、用例、接口文档与最近使用记录默认相互隔离，避免误混用。
- 激活环境要求：每个项目默认只保留一个激活环境，用于请求发送与变量解析。

## 10. 待确认项

- 是否需要在本期支持 OpenAPI 3.1、YAML 导入或更多规范变体。
- 环境配置是否需要导入导出、复制环境或批量切换变量启停能力。
- 是否需要提供全局公共变量，并在项目变量之上增加覆盖规则。

## 11. 当前假设

- 当前默认以本地单用户工具为目标，不引入登录和权限体系。
- 本期以本地数据持久化为主，暂不接入云端同步和团队协作能力。
- 当前默认以“项目”为主要工作区单元，接口文档、环境、用例和请求历史默认随项目隔离。
- 每个项目至少需要一个可用环境，且 `BaseUrl` 作为环境一级字段单独维护，而不是与普通变量完全等价。
- 每个项目默认仅允许一个激活环境，环境切换会整体切换当前请求上下文。
