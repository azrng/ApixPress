API 协作平台需求文档

一、项目概述

项目名称：API协作平台
项目目标：开发一个基础的API调试和管理工具，类似ApixPress风格，支持创建项目，Swagger导入、HTTP请求调试和用例保存

二、核心功能需求

2.1 Swagger导入功能

功能描述：支持导入Swagger/OpenAPI规范的API文档

详细需求：

1. 文件格式支持

  • 支持导入JSON格式的Swagger文档

  • 文件大小限制：≤ 20MB

2. 导入方式

  • 本地文件上传

  • URL导入（通过输入Swagger文档的URL地址）

  • 拖拽上传支持

3. 数据解析

  • 解析接口基本信息：名称、路径、方法、描述

  • 解析请求参数：查询参数、路径参数、请求头、请求体

  • 解析响应信息：状态码、响应示例

  • 解析数据模型（Schemas）

4. 数据展示

  • 以树形结构展示接口列表

  • 按API分组显示

  • 支持搜索和筛选接口

2.2 请求调试功能

功能描述：支持发送和调试HTTP请求

详细需求：

1. 请求配置区

  • 请求方法选择：GET、POST、PUT、DELETE

  • URL输入框：支持路径参数占位符（如 /api/users/:id）

  • 参数自动补全提示

2. 请求参数设置

  • 查询参数：支持键值对形式，支持批量编辑

  • 路径参数：自动从URL中提取并填充

  • 请求头：支持常用请求头预设，支持自定义

  • 请求体：

   ◦ Raw格式：支持Text、JSON、XML、HTML

   ◦ Form-data：文件上传支持

   ◦ x-www-form-urlencoded

   ◦ 二进制文件上传

  • Cookie管理：支持设置和自动携带Cookie

3. 请求发送与响应

  • 发送请求按钮，支持快捷键（Ctrl+Enter）

  • 响应状态显示：状态码、响应时间、响应大小

  • 响应内容展示：

   ◦ 格式化JSON/XML

   ◦ 下载文件

  • 响应头展示

  • 网络日志：请求详情、重定向信息

4. 高级功能

  • 环境变量支持：{{baseURL}}、{{token}}

  • 请求前/后脚本（基础版支持变量设置）

  • SSL证书忽略选项（开发环境用）

2.3 用例保存与管理

功能描述：保存和复用API请求用例

详细需求：

1. 用例保存

  • 保存当前请求所有配置：URL、方法、参数、请求体、请求头

  • 支持保存为用例集合（项目/分组）

  • 可添加用例描述和标签

  • 支持保存为模板

2. 用例管理

  • 用例列表展示

  • 用例分类/分组管理

  • 最近使用记录

3. 用例操作

  • 快速加载用例

  • 复制用例创建新请求

4. 数据存储

  • 本地存储方案：sqlite + 本地备份

  • 设置中心提供“存储”分组，可选择数据库存储目录；目录设置保存到 `Data/app-shell-settings.json`，数据库文件名固定为 `ApixPress.db`，重启后生效。

  • 设置中心支持一键清空系统数据，执行前会弹出二次确认；该操作会删除所有项目、接口、用例、环境配置和请求历史记录。

三、开发与发布说明

1. 桌面端主程序 `ApixPress` 默认按 `win-x64` 自包含单文件模式发布；更新流程已合并到主程序的隐藏更新模式中，不再发布独立更新器 exe。

2. 主程序默认配置与数据库初始化脚本已内嵌到程序集：

  • 内嵌默认配置：`appsettings.json`

  • 内嵌迁移脚本：`Data/Migrations/001_Initial.sql`

3. 若发布目录旁额外提供 `appsettings.json`，程序会优先使用内嵌默认配置，再叠加外部文件覆盖，便于按环境调整更新源或本地路径。

4. GitHub Actions 发布工作流位于 `.github/workflows/release.yml`，当前只发布主程序，并将主程序发布目录压缩为更新包。
